# 提案：实时书写（贴点分段二次曲线 + 末端对齐报点）

## 背景（Context）
当前工程已经具备 OpenGL ES 笔迹渲染能力与触摸输入采集，并存在“实时预览（live stroke）”的 JNI 通道，但在使用“拟合 + 重采样”的路径时，会出现轨迹与用户真实笔迹不贴合的问题（拐角被磨平、局部超调、节奏信息被均匀化），详见：
- `c:\Documents\Myproject\Android\HelloWorld\BezierQ&A.md#L130-158`

同时，实时预览如果末端不对齐“实际报点末端”，会造成书写时的主观时延感更明显；相反，末端对齐真实报点能显著改善输入到光子（input-to-photon）的体感，即使笔身存在轻微平滑也不影响感知。

因此需要新增并规范一套“实时书写”能力：用户可直接在画布上书写，实时预览始终渲染到报点末端，并且最终落笔后的渲染轨迹与用户真实轨迹贴合（避免全局拟合带来的失真）。

## 目标（Goals）
- 支持用户在画布上实时书写：按下开始、移动增量显示、抬起完成落笔
- 实时预览的渲染轨迹末端必须对齐“实际报点末端”（最后一点严格对齐）
- 最终落笔后的渲染轨迹与用户真实轨迹尽可能贴合，遵循“原始数据与渲染数据分离 + 局部贴点分段二次曲线 + 可控防抖”的思路
- 实时预览与最终落笔使用同一条几何生成策略，避免“预览像一条线，落笔变成另一条线”的割裂感

## 非目标（Non-Goals）
- 不引入新的笔刷材质、纹理、复杂端帽样式或抗锯齿方案
- 不在本次改动中追求多指书写、橡皮擦、选择/变形等编辑能力
- 不在本次提案中重做渲染架构（仍保持单次 instanced draw 的主路径）
