# 需求（Requirements）

## 功能需求（Functional Requirements）
- [ ] 用户可以在 `StrokeGLSurfaceView` 画布上直接书写，笔迹在按下后立刻出现，移动时连续增长，抬起后固定为最终笔迹
- [ ] 原始报点（真值）与渲染点序列必须分离：任何防抖/平滑只能作用在渲染点序列上，不允许回写修改原始报点
- [ ] 实时预览必须渲染到“实际报点的末端”：
  - [ ] 每次预览更新时，渲染点序列的最后一个点必须与最新原始报点相同（世界坐标一致）
  - [ ] 抬起时最终笔迹的末端必须与抬起时的最后报点对齐
- [ ] 放弃“全局拟合 + 重采样”，改为“局部贴点”的分段二次曲线生成策略，并支持“两段曲线”实时预览：
  - [ ] 已绘制固定段（Committed）：已稳定的历史部分，不再回滚重算
  - [ ] 末端可回滚段（Tail）：最近 K 个点参与滚动回修，K 可调（用于降低“修正感”）
  - [ ] 两段都必须输出曲线（非直连折线），且在连接处位置/方向尽量连续
  - [ ] 贴点分段二次曲线的基本规则：
    - [ ] 段连接点使用相邻点的中点
    - [ ] 非尾段的控制点优先取渲染锚点（来自原始点或轻量去噪/平滑后的点）
    - [ ] 尾段控制点必须按末端切线构造（不依赖未知的“下一个点”），同时保持末点严格对齐最新原始报点
    - [ ] 每段仅受局部点影响，避免曲线外扩与角点被抹平
- [ ] 增加尾段稳定化平滑算法（仅作用于 Tail 与渲染锚点）：
  - [ ] 平滑强度可控，并带“保角门控”，避免把折笔磨圆
  - [ ] 当点从 Tail 进入 Committed 时，使用平滑后的渲染锚点冻结为稳定点（原始报点仍保留不变）

## 一致性与验收标准（Acceptance Criteria）
- [ ] 端点一致性：实时预览与最终落笔都满足“最后渲染点 == 最后原始报点”
- [ ] 贴合性：在包含明显拐角的输入上，最终笔迹不出现明显超调/外扩，拐角形态可被肉眼辨识（不被磨成圆角）
- [ ] 观感一致性：Tail 回修不产生明显“回拉/跳动”，K 调大可减弱修正感但不引入明显时延
- [ ] 可回归：提供可复现输入（使用现有回放数据或录制数据），可在单元测试/回放界面中验证端点一致性与关键几何约束

## 用户场景（User Scenarios）
1. 用户按下并快速写一个“折笔”形状（如“V/7”）：实时预览末端紧跟手指位置，抬起后拐角仍清晰可见
2. 用户慢速写直线：防抖去除硬件高频抖动，线条稳定但末端仍对齐最新报点
3. 用户缩放/平移视图后继续书写：输入采集仍在世界坐标系工作，笔迹与视图变换一致
