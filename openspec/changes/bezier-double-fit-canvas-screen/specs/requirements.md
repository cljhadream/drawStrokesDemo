# 需求（Requirements）

## 术语
- **报点**：触控/手写事件序列的离散采样点，包含 `(x, y, timestamp, eventType)`
- **eventType**：至少包含 `DOWN`、`MOVE`、`UP`（允许扩展如 `CANCEL`、`POINTER_DOWN` 等，但本界面以单指书写为主）
- **归一化画布**：以当前画布宽度为基准，将坐标缩放到“宽度=1000px”的坐标系
- **二次贝塞尔段**：以 `P0(起点) -> P2(终点)` 为端点，`P1` 为控制点的二次贝塞尔曲线
- **拟合（Fit）**：将报点序列转换为一组二次贝塞尔段，并可从曲线中重采样得到新的点序列

## 功能需求（Functional Requirements）
- [ ] MainActivity 增加一个按钮，可跳转到“贝塞尔回放界面（Canvas）”
- [ ] 新界面内置一段固定报点 JSON（包含完整 down-move-...-up 事件），打开页面即可执行处理流程并渲染结果
- [ ] 处理流程必须严格按以下顺序执行：
  - [ ] 以画布宽度为基准缩放到“宽度=1000px”的归一化坐标系
  - [ ] 在归一化坐标系对报点执行一次二次贝塞尔拟合，得到曲线，并从曲线上重采样生成“拟合点序列 A”
  - [ ] 将“拟合点序列 A”按当前画布宽度放大回屏幕坐标系，得到“点序列 B”
  - [ ] 在屏幕坐标系对“点序列 B”再次执行二次贝塞尔拟合，得到最终曲线
  - [ ] 使用 Canvas 将最终曲线绘制到屏幕
- [ ] 绘制需提供对照能力：
  - [ ] 可选显示原始报点散点（用于确认曲线是否穿过关键点）

## 一致性需求：拐点与报点约束（必须满足）
- [ ] 拟合结果必须通过所有 `DOWN/UP` 端点
- [ ] 必须保留拐点（用户意图的折笔点），不得被双次拟合抹平
- [ ] “二次拟合链路”中必须保留原始报点作为锚点（anchor），避免“拟合后点序列”丢失角点信息

## 建议实现约束（用于指导实现，非强制）
- [ ] 拟合采用“插值式分段二次贝塞尔”：每段曲线端点使用原始点（或锚点），保证曲线至少经过所有锚点
- [ ] 角点检测使用阈值策略（例如基于相邻向量夹角、速度变化），角点处将其强制作为分段边界并降低平滑
- [ ] 重采样必须包含所有锚点（至少包含每段端点与角点），防止第二次拟合进一步平滑

## 输入格式（建议）
- [ ] 直接写死一段固定报点 JSON
- [ ] 每个点至少包含：`x`, `y`, `timestamp`, `eventType`
- [ ] 至少支持 `DOWN/MOVE/UP` 三类事件；遇到 `CANCEL` 时结束当前笔迹

## 用户场景（User Scenarios）
1. 用户粘贴一段手写报点（含明显拐弯），界面能画出最终拟合曲线，并可与原始报点对照，确认拐弯未被抹平
2. 用户切换不同画布尺寸（旋转屏幕或改变窗口大小），归一化到 1000px 后的拟合效果保持一致
3. 用户输入多段笔迹（多次 down-up），系统能分别拟合并绘制每段曲线，不跨笔迹连线
