# 提案：双次二次贝塞尔拟合回放界面（Canvas）

## 背景（Context）
当前我们需要验证一条“报点 -> 坐标归一化 -> 二次贝塞尔拟合 -> 坐标放大回屏幕 -> 再次二次贝塞尔拟合 -> Canvas 渲染”的流程，用来对比不同拟合策略在“缩放/重采样/再次拟合”链路上的几何稳定性。

现有问题集中在：**二次贝塞尔拟合后曲线不经过实际报点**，当拟合链路执行两次时，路径会被进一步平滑，导致写字的“拐弯/折笔”特征被抹掉，轨迹与用户意图不一致。

为了让问题可复现、可调参、可对比，需要新增一个独立界面，能够输入完整事件序列（down/move/.../up），按指定流程计算并在屏幕上绘制“最终拟合曲线”，并提供“原始报点轨迹”与“拟合结果”的可视化对照。

## 目标（Goals）
- 新增一个可从 MainActivity 进入的调试界面，用于输入/回放一段报点数据并在 Canvas 上绘制结果
- 严格实现指定处理链路：宽度归一化到 1000px -> 二次拟合 -> 放大回屏幕 -> 二次拟合 -> 绘制
- 解决“曲线不经过实际报点、拐弯被抹平”的核心问题：最终输出必须保留拐点与端点，且路径应尽可能经过原始报点

## 非目标（Non-Goals）
- 不将该流程接入现有 OpenGL ES 笔迹渲染引擎（该界面仅用于验证/调试）
- 不追求极致性能优化（输入规模以调试可用为主）
- 不引入复杂笔刷材质、压感宽度、抗锯齿等效果（以几何拟合与轨迹一致性为主）
