# 需求（Requirements）

## 术语与指标
- **长帧（Long Frame）**：单帧耗时超过当前刷新周期（60Hz 为 16.6ms，90Hz 为 11.1ms，120Hz 为 8.3ms）
- **frame pacing**：帧耗时分布与抖动情况（p50/p95/p99、方差、连续掉帧段长度）
- **input-to-photon**：触控事件时间戳到对应笔迹像素首次出现在屏幕上的延迟（p50/p95/p99）
- **高负载场景**：满足任一条件即可判定为高负载：
  - 近 1 秒窗口内长帧比例超过阈值
  - UI/GL 线程出现连续掉帧段
  - GPU/合成压力导致 swap 阻塞显著增加（以实际采样指标为准）

## 功能需求（Functional Requirements）
- [ ] 录制并输出关键性能指标：渲染帧耗时分布、连续掉帧段、input-to-photon 分布、每帧 CPU/GPU 主要阶段耗时
- [ ] 支持在运行时开启/关闭“性能 HUD”，展示上述指标的实时摘要
- [ ] 引擎具备“自适应质量档位”，可根据实时帧时与掉帧情况自动切换，并支持手动强制档位以便定位
- [ ] 自适应策略切换必须平滑：禁止频繁抖动切换（需滞回/冷却时间）
- [ ] 在“用户正在书写”期间优先保障跟手；松手后允许进行补偿性重建/高质量回填（若采用交互期降级渲染）
- [ ] 必须维持当前核心约束：
  - 单次 `glDrawArraysInstanced` 绘制全部笔迹
  - 每条笔迹独立颜色与透明度
  - 缩放逻辑在 shader 中完成（笔宽随缩放校正，保持物理宽度不变）

## 性能目标（Performance Targets）
说明：目标以“可感知改善 + 可回归”为原则，具体阈值以设备矩阵与采样数据为准。

- [ ] 常规场景（无视频/无共享屏幕）：
  - 60Hz 设备：书写期间 p95 帧耗时 ≤ 16.6ms，p99 ≤ 20ms
  - input-to-photon：p50 ≤ 35ms，p95 ≤ 60ms
- [ ] 高负载场景（视频播放或会议共享屏幕开启）：
  - 60Hz 设备：书写期间 p95 帧耗时 ≤ 33ms，连续掉帧段显著减少（以对比基线版本为准）
  - input-to-photon：p95 明显下降（以对比基线版本为准），并避免出现“明显断续式刷新”
- [ ] 资源约束：
  - 自适应降级必须降低至少一项可度量的渲染成本（CPU、GPU、带宽或同步阻塞）
  - 不引入不可控的内存增长（需有上限与回收策略）

## 用户场景（User Scenarios）
1. 用户在 1080p/4K 视频播放时手写标注，笔迹跟手且抖动显著少于基线版本
2. 用户在在线会议中共享屏幕并手写，仍能连续书写不“断笔”，且松手后画面可逐步回到高质量
3. 用户在不同缩放倍率下书写与拖拽缩放，笔宽保持一致，笔迹颜色/透明度正确，且降级策略不破坏交互一致性
