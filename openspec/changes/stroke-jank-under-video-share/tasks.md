# 实施任务（Implementation Tasks）

- [ ] 1. 建立可观测性与复现场景
  - [ ] 1.1 增加输入与渲染关键阶段打点（Kotlin + JNI + C++）
  - [ ] 1.2 增加可选性能 HUD（帧耗时、掉帧段、input-to-photon 摘要）
  - [ ] 1.3 制定复现脚本：视频播放/会议共享屏幕 + 连续书写 + 缩放拖拽
  - [ ] 1.4 用 Perfetto 抓取基线版本数据，形成对比报告（p50/p95/p99）

- [ ] 2. 优化帧调度与渲染触发
  - [ ] 2.1 将 GLSurfaceView 切换为按需渲染，并用 vsync 驱动 requestRender
  - [ ] 2.2 增加渲染节流：高负载下稳定降到 30fps（或设备刷新率的一半）
  - [ ] 2.3 校验触控路径：UI 线程不阻塞、无频繁分配、无跨线程等待

- [ ] 3. 优化输入队列与增量几何处理
  - [ ] 3.1 引入无锁/低锁输入缓冲，按帧批量消费触控点
  - [ ] 3.2 将贝塞尔拟合与切分改为增量处理，避免每帧全量重算
  - [ ] 3.3 维护每条笔迹包围盒与 dirty region，为局部更新做准备

- [ ] 4. 优化 GPU 数据更新与同步
  - [ ] 4.1 拆分静态与动态笔迹数据，减少静态区域的重复上传
  - [ ] 4.2 为动态 SSBO/VBO 引入双/三缓冲轮转，降低隐式 stall
  - [ ] 4.3 通过 fence 做显式同步与上限控制，避免 swap/更新阻塞扩散

- [ ] 5. 实现自适应质量与（可选）交互期预览层
  - [ ] 5.1 定义质量档位与可调参数（细分密度、AA、join 策略等）
  - [ ] 5.2 实现基于窗口指标的降级/恢复策略（含滞回与冷却时间）
  - [ ] 5.3 （可选）实现交互期轻量预览通道，松手后矢量回填重绘

- [ ] 6. 验证与回归
  - [ ] 6.1 在设备矩阵上对比基线与优化版本（普通/高负载场景）
  - [ ] 6.2 验收指标：掉帧段显著减少、input-to-photon 尾延迟下降
  - [ ] 6.3 回归检查：单次 instanced draw、缩放笔宽、颜色/透明度一致性
